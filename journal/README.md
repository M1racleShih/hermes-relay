# Journal

Journal 不是日报，也不是每次学习都必须写的长文。它只记录 commit message 不适合承载的学习过程：探索路径、未解决问题、被否定的假设、下一次接续点。

如果一次学习已经形成了高质量笔记，并且 commit message 清楚记录了目标、变更和验证，可以不写 journal。

需要 journal 时再新建：

```bash
make new-log TITLE=tool-registry-first-pass
```

短 journal 优先回答：

- 今天切哪个源码面？
- 读了哪些文件/函数？
- 得到什么结论？
- 验证了什么？
- 下一次从哪里继续？
